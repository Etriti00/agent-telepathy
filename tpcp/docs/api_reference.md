# TPCP API Reference

## `TPCPNode` (`tpcp-core` & `tpcp-ts`)
The primary gateway for interacting with the TPCP protocol.

### Initialization
```python
node = TPCPNode(
    identity: AgentIdentity, 
    host: str = "127.0.0.1", 
    port: int = 8000, 
    adns_url: Optional[str] = None
)
```
- **`identity`**: The cryptographic identity generated by the `AgentIdentity` schema.
- **`host` / `port`**: Local binding for the WebSocket server.
- **`adns_url`**: Optional. Binds the node to the global Agent DNS lookup server.

### Core Methods

#### `async register_peer(identity: AgentIdentity, address: str)`
Registers a peer in the local routing table, tracking their `agent_id` against their connection string.

#### `async send_message(target_id: UUID, intent: Intent, payload: Payload)`
Routes a payload to a specific peer. 
*Note:* Automatically signs the payload and enforces DLQ (Dead-Letter Queue) exponential back-off retries if the network drops.

#### `async start_listening()`
Boots the background threaded WebSocket listener. If `adns_url` was provided, actively hooks the remote relay and broadcasts its presence.

#### `async broadcast_discovery(seed_nodes: list[str])`
Pings an active connection to arbitrary seed WebSocket arrays or the A-DNS letting remote swarms detect the node dynamically.

---

## The Handlers (Intents)
TPCP categorizes wire arrays into specific intent architectures.

*   `Intent.HANDSHAKE`: Node initialization and discovery polling.
*   `Intent.STATE_SYNC`: Explicit routing pointing to internal `CRDTSyncPayload`s to merge distributed graphs.
*   `Intent.STATE_SYNC_VECTOR`: Overrides standard LLM context windows by directly piping raw float embedding schemas into internal memory banks logic.
*   `Intent.TASK_REQUEST`: (Future) Dedicated routing targeting executor swarms.
